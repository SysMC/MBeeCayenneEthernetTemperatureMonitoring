MBeeCayenneEthernetTemperatureMonitoring_bb
Данный проект предназначен для реализации системы сбора данных с автономных датчиков температуры, 
передающих свои показания по радиоканалу с помощью модулей MBee с программным обеспечением SerialStar.
Полученные данные передаются в интернет через шлюз, реализованный на Arduino + Ethernet Shield +
Wireless Shield + MBee-868-2.0. Данные накапливаются и отображаются с помощью сервиса mydevices
Предварительно необходимо зарегистрироваться на сайте https://mydevices.com/. Создать новый проект.
Получить ключ проекта. Добавить этот ключ в sketch. 

Настройка модуля MBee для шлюза:

+++		//вошли в командный режим
AT RE		//возврат к заводским настройкам
AT AP 1		//включили пакетный режим
AT CN		//сохранили настройки и вышли из командного режима


Настройка модуля MBee для использования в качестве автономного датчика температуры:
(в данном проекте, шлюз настроен на прием данных от датчиков с адресами 0x06, 0x07, 0x08)

+++		//вошли в командный режим
AT RE		//возврат к заводским настройкам
AT SM 4		//установили циклический режим сна
AT SP 1000	//установили период сна 10 секунд
AT MY 6		//Установили собственный ID устройства 
AT L2 0		//Отключаем неиспользуемые выводы
AT L3 0
AT L5 0
AT R1 0
AR R2 0
AT R4 0
AT R6 0
AT CN		//сохранили настройки и вышли из командного режима


Модули, настроенные для использования в качестве автономного датчика температуры устанавливаем на
MB-Tag, подключаем батарейку. 

Открываем приложение mydevices на смартфоне или компьютере. Добавляем в нем widget, привязанный к 
виртуальному порту V1. Это будет индикаторы температуры устройства с ID 0x06. К виртуальному порт V2
привязываем widget, который будет отображать напряжение питания данного устройства (напряжение на батарейке).
Для устройства с адресом 0x07 используем виртуальные порты V3 для температуры и V4 для батарейки.
Для устройства с адресом 0x08 используем виртуальные порты V5 для температуры и V6 для батарейки.
